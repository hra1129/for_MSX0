MSX-BASIC は、変数に最初に代入したタイミングで変数の領域が確保されます。
変数の領域は、
	[型番号][変数名 2byte][値 (型によって長さが異なる]
な領域です。これが連なる可変長配列となっています。

変数は、どんどん後ろに追加される形で増えていきますが、
数式の中でその変数が使われているとき、この可変長配列を頭から順次検索して
その変数を見つけてから値を参照する処理になっています。
Z80 では、この探索もそれなりに時間がかかってしまいます。

従って、最初に代入された変数ほど高速に利用できることになります。


100 DEFINTA-Z:A=0:B=0:C=0:D=0:E=0:F=0:G=0:H=0:I=0:J=0:K=0
110 TIME=0:FORA=0TO10000:B=C+1:NEXT:PRINT TIME
120 TIME=0:FORA=0TO10000:B=K+1:NEXT:PRINT TIME

OpenMSX の A1GTエミュレーションで 

271
297

の表示となっている。結構違ってますね。
